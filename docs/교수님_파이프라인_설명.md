# AIPA 기술 파이프라인 단계별 설명

> 첨부 이미지 참고:
> - 이미지 1: AIPA 통합 시스템 및 파인튜닝 파이프라인 설계도
> - 이미지 2: AIPA 페르소나 임베딩 모델 아키텍처

---

## 전체 흐름 요약

```
설문지 PDF 업로드
    → 문항 자동 추출 (Claude API)
    → AI 페르소나 생성 (KOSIS 통계 기반)
    → 점수 예측 (임베딩 모델)
    → 트렌드 보정 (RAG)
    → 이유 문장 생성 (0.5B 파인튜닝 모델)
    → 인간 편향 시뮬레이션
    → 캘리브레이션 (통계 보정)
    → 최종 결과 출력
```

## 파이프라인 단계 1: 학습 데이터 생성 (Knowledge Distillation)

### 무엇을 하는가

대형 AI(Claude API)에게 "이 페르소나가 이 제품/설문을 보면 어떻게 반응할까?"를 반복 질문하여 학습 데이터를 수집합니다.

### 왜 필요한가

자체 모델을 학습시키려면 정답 데이터가 필요합니다. 실제 사람을 모집해서 데이터를 모으면 비용이 수백만 원 이상 들지만, 대형 AI의 판단 능력을 활용하면 약 5만 원(API 호출 비용)으로 충분한 양의 고품질 데이터를 확보할 수 있습니다.

### 구체적 과정

1. 16개 카테고리(식품, 화장품, 광고, 정책 등) × 다양한 시나리오 124개를 준비
2. 40명의 다양한 페르소나(10대~60대, 남녀, 직업 20종)를 설정
3. Claude API에게 "이 페르소나가 이 자극물을 보면 각 평가축에 몇 점을 줄까?"를 질문
4. 2,925건의 평가 데이터 + 1,402건의 설문 응답 데이터 = 총 4,327건 수집

### 데이터 1건의 예시

```
질문: "50대 남성 임원(품질 중시, 건강 관심)이 유니클로 에어리즘 티를 보면?"
Claude 답변:
  - 디자인호감도: 62점 ("오버사이즈 핏이 유행이지만 임원급으로서는 캐주얼함이 다소 아쉽다")
  - 구매의향: 58점 ("가격은 합리적이지만 품질을 중시하는 성향상 프리미엄 브랜드를 선호")
  - 가격적절성: 75점 ("19,900원은 품질 대비 합리적이라고 판단")
```

### 이 과정이 가지는 의미

이것이 "Knowledge Distillation(지식 증류)"입니다. 큰 모델(Claude, 수천억 파라미터)의 판단 능력을 작은 모델(5만 파라미터)로 옮기는 기법으로, 학습 이후에는 Claude 없이도 독립적으로 점수 예측이 가능해집니다.

## 파이프라인 단계 2: 임베딩 모델 학습 (점수 예측)

> 이미지 2 참고: "AIPA 페르소나 임베딩 모델 아키텍처"

### 무엇을 하는가

페르소나 속성(연령, 성별, 직업, 성격)과 평가 대상(카테고리, 평가축)을 입력하면, 0~100점 사이의 점수를 예측하는 경량 모델입니다.

### 모델 아키텍처 (이미지 2 각 계층 설명)

**[1] 입력 계층 (Input Layer)**

문자열 데이터를 숫자 인덱스로 변환합니다.
- 연령: "10대"~"60대+" (6개 카테고리)
- 성별: male, female, non-binary (3개)
- 직업: 고등학생, 대학생, 직장인, 임원 등 (44개)
- 특성: "가성비 중시", "SNS 활발" 등 (90개, 1인당 최대 5개)
- 카테고리: 식품, 화장품, 광고 등 (16개)
- 평가축: 호감도, 구매의향, 가격적절성 등 (50개)

**[2] 임베딩 & 집계 계층 (Embedding & Aggregation Layer)**

각 인덱스를 의미 있는 숫자 벡터(수치 목록)로 변환합니다.
- 연령 → 24차원 벡터
- 성별 → 12차원 벡터
- 직업 → 24차원 벡터
- 특성 → 24차원 벡터 × 5개 → Attention 메커니즘으로 중요도 가중합하여 1개의 24차원 벡터로 집계
- 카테고리 → 24차원 벡터
- 평가축 → 24차원 벡터

Attention의 역할: 특성이 5개 있을 때 상황에 따라 중요한 특성에 더 높은 가중치를 부여합니다. 예를 들어 식품을 평가할 때는 "가성비 중시"에 높은 가중치, 패션을 평가할 때는 "트렌드 민감"에 높은 가중치가 자동으로 학습됩니다.

**[3] 인코더 계층 (Encoder Layer)**

- Persona Encoder: 연령(24) + 성별(12) + 직업(24) + 특성(24) = 84차원을 concat → Linear(84→64) → ReLU → Dropout(0.2) → Linear(64→24) → "이 사람은 어떤 사람인가"를 나타내는 24차원 벡터
- Context Encoder: 카테고리(24) + 평가축(24) = 48차원을 concat → Linear(48→64) → ReLU → Linear(64→24) → "무엇을 평가하는가"를 나타내는 24차원 벡터

**[4] 예측 계층 (Predictor Layer)**

- 페르소나 벡터(24) + 컨텍스트 벡터(24) = 48차원을 concat
- Linear(48→64) → ReLU → Dropout(0.2) → Linear(64→32) → ReLU → Linear(32→1) → Sigmoid
- Sigmoid 출력(0~1)에 100을 곱하여 최종 0~100점 반환

### 학습 설정

| 항목 | 값 |
|------|-----|
| 파라미터 수 | ~50,000개 (약 1MB) |
| 손실함수 | MSE (Mean Squared Error) |
| 옵티마이저 | Adam (lr=0.001) |
| 스케줄러 | Cosine Annealing |
| 에포크 | 100 |
| 배치 사이즈 | 64 |
| Train/Eval | 90% / 10% |
| 평균 오차 | ~8.7점 (100점 만점) |
| 추론 속도 | 0.01초 (CPU) |

## 파이프라인 단계 3: 파인튜닝 모델 학습 (이유 문장 생성)

> 이미지 1 참고: "2. 학습 계층" → "파인튜닝 모델 학습"

### 무엇을 하는가

임베딩 모델이 예측한 점수(예: "구매의향 72점")를 입력으로 받아서, "왜 이 점수가 나왔는지" 이유 문장을 생성합니다. 이 모델은 스스로 점수를 판단하지 않습니다. 이미 확정된 점수에 설명만 붙이는 역할입니다.

### 왜 별도 모델이 필요한가

- 임베딩 모델은 "72점"이라는 숫자만 출력하고, 이유는 말하지 못합니다.
- 그렇다고 매번 Claude API에 이유를 물어보면 비용이 발생합니다.
- 작은 LLM(0.5B)을 파인튜닝하면 CPU에서 무료로 이유를 생성할 수 있습니다.

### 두 가지 버전

| | AIPA-Eval (7B) | AIPA-Reasoning (0.5B) |
|---|---|---|
| 베이스 모델 | Qwen 2.5-7B | Qwen 2.5-0.5B |
| 역할 | 점수 + 이유 동시 생성 | 이유만 생성 |
| 입력 | 페르소나 + 자극물 | 페르소나 + 자극물 + **점수** |
| GPU 필요 | O (RTX 4070 Ti) | X (GGUF 변환 후 CPU) |
| 용도 | 연구/오프라인 (초기 버전) | **실서비스 배포** |

실서비스에서는 0.5B 모델만 사용합니다. 7B 모델은 개발 초기에 "점수+이유를 동시에 생성"하는 용도로 만들었으나, 임베딩 모델이 점수 예측을 대체하게 되면서 0.5B 모델이 이유 생성만 담당하는 구조로 발전했습니다.

### 학습 데이터 변환

원본 데이터의 "점수"를 출력에서 입력으로 이동시킵니다.

```
변환 전: 입력(페르소나+자극물) → 출력(점수+이유)
변환 후: 입력(페르소나+자극물+점수) → 출력(이유만)
```

점수 예측은 임베딩 모델이 0.01초 만에 해결했으므로, 0.5B 모델에게는 가벼운 작업(이유 생성)만 맡기는 것입니다.

### 파인튜닝 기법: QLoRA

전체 모델을 처음부터 학습하면 GPU 메모리와 시간이 많이 필요합니다. 대신 LoRA(Low-Rank Adaptation) 기법을 사용합니다.

- 원본 모델의 가중치를 고정(freeze)하고, 작은 어댑터 행렬만 학습
- 전체 파라미터의 약 1%만 학습 → GPU 메모리/시간 대폭 절약
- 4bit 양자화(NF4)로 모델을 압축하여 로드 → QLoRA
- Transformer의 Attention(Q,K,V,O)과 FFN(gate, up, down) 7개 레이어에 어댑터 부착
- r=16, alpha=32, dropout=0.05

### 배포 파이프라인

```
학습 완료
  → LoRA 어댑터 + 베이스 모델 병합 (merge_and_unload)
  → GGUF 형식 변환 (4bit 양자화, q4_k_m)
  → Cloud Run 배포 (CPU 전용, GPU 불필요)
  → 최종 크기: 약 300~400MB
  → 응답 시간: 1~2초
```

## 파이프라인 단계 4: RAG 트렌드 보정

### 무엇을 하는가

네이버 데이터랩 검색 트렌드와 KOSIS 통계 데이터를 벡터 DB에 저장해두고, 평가 시 관련 트렌드를 검색하여 점수를 보정합니다.

### 왜 필요한가

같은 "유기농 식품"이라도 2024년과 2026년의 소비자 관심도는 다릅니다. 임베딩 모델은 학습 시점의 패턴만 알고 있으므로, 현재 트렌드를 반영하는 보정이 필요합니다.

### 구체적 과정

1. **데이터 수집**: 네이버 검색 트렌드, 쇼핑 트렌드, KOSIS 인구통계 데이터를 주기적으로 수집
2. **벡터 변환**: ko-sroberta-multitask 모델로 텍스트를 벡터로 변환
3. **ChromaDB 저장**: 벡터를 ChromaDB(벡터 데이터베이스)에 인덱싱
4. **검색 & 보정**: 평가 요청 시 관련 트렌드를 유사도 검색하여, 긍정 트렌드면 관련 축 점수를 +3~10점 보정

### 보정 예시

```
"건강식품" 평가 요청이 들어옴
  → ChromaDB에서 "건강식품" 관련 트렌드 검색
  → "최근 건강식품 검색량 30% 증가" 문서 발견
  → 긍정 키워드 "증가, 성장" 감지
  → 호감도, 구매의향 등 트렌드 민감 축에 +5점 보정
```

## 파이프라인 단계 5: 실서비스 동작 흐름

> 이미지 1 참고: "4. 배포 및 실서비스 파이프라인"

### 평가 요청이 들어왔을 때

```
Step 1: 임베딩 모델로 점수 예측 (0.01초)
  "20대 여성 대학생" + "식품" + "구매의향" → 72점
  → 이 시점에서 점수 확정

Step 2: RAG 트렌드 보정 (0.1초)
  ChromaDB에서 관련 트렌드 검색
  → 최근 건강식품 인기 상승 → 구매의향 72점 → 77점으로 보정

Step 3: 이유 문장 생성 (1~2초, 3단계 폴백)
  1순위: 자체 0.5B 모델 (무료, CPU)
    → "가성비를 중시하는 대학생이라 합리적 가격에 높은 구매의향을 보임"
  2순위: Claude API (유료, 고품질)
    → 0.5B 모델 실패 시 사용
  3순위: 템플릿 (무료, 즉시)
    → "20대 대학생의 관점에서 구매의향이 긍정적으로 평가됩니다"

최종 출력:
  {"name": "구매의향", "score": 77, "reasoning": "가성비를 중시하는..."}
```

### 이유 생성이 실패하더라도

점수는 임베딩 모델이 이미 확정했으므로 정상 반환됩니다. 이유 문장만 품질이 떨어질 뿐(템플릿으로 대체), 핵심 기능(점수 예측)은 영향받지 않습니다.

## 파이프라인 단계 6: 인간 편향 시뮬레이션

### 무엇을 하는가

AI가 생성한 "완벽한" 응답에 실제 사람이 설문에 응답할 때 나타나는 편향을 적용합니다.

### 8가지 편향

| 편향 | 설명 | 예시 |
|------|------|------|
| 극단 회피 | 양 끝 답변을 피하고 중간 선호 (한국인 특성) | 1점/5점 대신 2점/4점 선택 |
| 묵종 편향 | "그렇다" 쪽에 동의하는 경향 | 긍정 방향으로 약간 이동 |
| 사회적 바람직성 | 환경/건강 질문에 더 긍정적 답변 | "분리수거 합니까?" → "매우 그렇다" |
| 피로 효과 | 설문이 길어지면 같은 답 반복 | 후반부 문항에 일괄 "보통" |
| 무응답 | 민감 질문(소득/정치) 스킵 | 소득 질문 → 무응답 처리 |
| 또래 효과 | 같은 연령대끼리 비슷한 응답 | 20대끼리 응답 분포 수렴 |
| 시간대 효과 | 야간 충동적, 월요일 부정적 | 심야 응답 시 극단값 증가 |
| 응답 노이즈 | 같은 사람도 매번 ±5~10점 변동 | 동일 질문 재응답 시 미세 차이 |

## 파이프라인 단계 7: 캘리브레이션 (통계 보정)

### 무엇을 하는가

생성된 AI 패널의 인구통계 분포가 실제 한국 인구 분포와 일치하도록 가중치를 조정합니다.

### 왜 필요한가

AI가 생성한 패널 50명의 분포가 "20대 30명, 50대 20명"이라면 실제 인구 비율과 다릅니다. 실제 여론조사 기관이 사용하는 사후 보정(post-stratification) 방법론을 적용하여 결과의 대표성을 확보합니다.
