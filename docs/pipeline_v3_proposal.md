# AIPA - 기술 파이프라인 (예창패 제안서용)

---

## 한 줄 요약

> AI 가상 패널이 실제 사람처럼 설문에 응답하고, 제품/광고에 대한 소비자 반응을 예측하는 시뮬레이션 플랫폼

---

## 서비스 구성

```
AIPA 모바일 앱 (Android)
  │
  ├── [설문 시뮬레이션] AI 패널 N명이 설문에 응답
  │
  └── [AI 채팅] 원하는 페르소나 1명과 1:1 대화
```

---

## 핵심 기능 1: 설문 시뮬레이션

### 사용 흐름 (유저 관점)

```
① 패널 선택
   유저가 "MZ세대 15명" 또는 "전문가 5명" 같은 패널 선택
   또는 직접 연령/성별/직업/성격을 설정

② PDF 설문지 업로드
   설문지 PDF를 올리면 자동으로 질문과 보기를 추출
   (테이블 형태의 설문도 인식)

③ 시뮬레이션 실행
   AI 패널이 각자의 관점에서 설문에 응답
   진행률 실시간 표시 (10% → 50% → 90% → 완료)

④ 결과 확인
   질문별 응답 분포 차트 (막대그래프)
   개별 페르소나의 선택 + 이유
   CSV 파일로 내보내기 가능
```

### 내부 기술 흐름

```
[1단계] 페르소나 생성 (0.1초)
   - 통계청(KOSIS) 실제 한국 인구통계 데이터 기반
   - 연령/성별/직업 분포가 실제 통계와 일치하도록 생성
   - 조건부 확률 적용: "20대이면 학생 35%, 회사원 30%"
   - 연령대에 맞는 성격 특성 자동 부여
     (10~20대: 트렌디, 디지털 친숙 / 50대+: 건강 관심, 안전 중시)

[2단계] 반응 점수 예측 (0.01초)
   ★ 자체 개발 임베딩 모델 (22,862 파라미터)
   - 페르소나 속성(연령/성별/직업/성격)을 수치화
   - "이 사람이 이 질문에 얼마나 긍정적일까?"를 0~100점으로 예측
   - 학습 데이터: 4,327건의 한국 소비자 반응 합성 데이터
   - 평균 오차: 8.7점 (100점 만점 기준)

[3단계] 트렌드 보정 (0.1초)
   - 네이버 데이터랩 검색 트렌드를 반영
   - "유기농" 검색량이 증가 추세면 → 관련 점수 상승
   - 같은 질문이라도 시기에 따라 다른 결과 (실제 사람처럼)

[4단계] 답변 생성 (3~10초)
   ★ 자체 학습 경량 AI 모델 (Qwen 2.5-0.5B)
   - 임베딩 모델이 예측한 점수에 맞는 보기를 선택
   - 선택 이유를 페르소나 관점에서 자연어로 생성
   - 예: 점수 78점 → "매우 그렇다" 선택
         → "카페에서 일하는 게 더 집중 잘 돼요"

[5단계] 인간 응답 편향 적용
   실제 설문조사 연구 기반 8가지 편향 시뮬레이션:
   - 극단 회피: 양 끝 답변 피하고 중간 선호 (한국인 특성)
   - 묵종 편향: "그렇다" 쪽에 동의하는 경향
   - 사회적 바람직성: 환경/건강 질문에 더 긍정적 답변
   - 피로 효과: 설문이 길어지면 같은 답 반복
   - 무응답: 민감 질문(소득/정치) 스킵
   - 또래 효과: 같은 연령대끼리 비슷한 응답
   - 시간대 효과: 야간 충동적, 월요일 부정적
   - 응답 노이즈: 같은 사람도 매번 ±5~10점 변동

[6단계] 캘리브레이션 (통계 보정)
   - 생성된 패널의 인구통계 분포를 목표에 맞게 가중치 보정
   - 실제 여론조사 기관이 사용하는 사후 보정 방법론 적용
```

## 핵심 기능 2: AI 채팅

### 사용 흐름 (유저 관점)

```
① 페르소나 선택
   기본 제공: AI 교수님, MZ 마케터, IT 개발자, 50대 주부
   또는 직접 만들기: 연령/성별/직업/성격/관심사 선택

② 파일 첨부 (선택)
   제품 기획서, 광고 이미지, 사업계획서 등 첨부 가능
   이미지는 AI가 직접 인식 (멀티모달)

③ 대화 목적 선택
   피드백 / 평가 / 개선안 / 점수 매기기 / 반론

④ 1:1 대화
   선택한 페르소나의 관점에서 답변
   대화 중 추가 파일 첨부 가능
   대화 내용 클립보드 복사 가능
```

## 기술 차별화

### 일반 서비스와의 비교

| 구분 | ChatGPT에 직접 물어보기 | AIPA |
|------|----------------------|------|
| 페르소나 | "20대라고 치고 답해줘" (상상) | 실제 인구통계 기반 생성 |
| 판단 | AI가 감으로 답함 | **자체 임베딩 모델**이 점수 예측 |
| 표현 | 외부 API가 전부 처리 | **자체 학습 AI**가 답변 생성 |
| 현실성 | AI 특유의 완벽한 답 | **8가지 인간 편향** 시뮬레이션 |
| 트렌드 | 학습 시점 지식만 | **실시간 검색 트렌드** 반영 |
| 통계 보정 | 없음 | **캘리브레이션** 적용 |
| 비용 | 매 질문마다 과금 | 설문/평가는 **자체 모델로 무료** |

### 자체 기술 스택

| 기술 | 설명 | 용도 |
|------|------|------|
| 페르소나 임베딩 모델 | 22,862 파라미터, 104KB | 소비자 반응 점수 예측 (0.01초) |
| 경량 LLM (0.5B) | 자체 파인튜닝, 2.8GB | 답변 선택 + 이유 생성 (3~10초) |
| 학습 데이터셋 | 4,327건 (평가 2,925 + 설문 1,402) | 한국 소비자 반응 합성 데이터 |
| 인간 편향 엔진 | 8가지 편향 모델 | 현실적 응답 시뮬레이션 |
| RAG 트렌드 보정 | 네이버 데이터랩 연동 | 시대적 맥락 반영 |

### 모델 독립성

```
외부 API 의존:  채팅(Claude API)만 사용
자체 모델 처리: 설문 시뮬레이션, 자극물 평가 전부 자체 모델
→ API 비용 대폭 절감, 외부 장애 시에도 핵심 기능 동작
```

## 부록: 자체 모델 개발 과정 상세

### 1. 임베딩 모델 (반응 점수 예측)

#### 모델이 하는 일

```
입력: "20대 여성 대학생, 가성비 중시" + "유기농 샐러드 배달 서비스" + "구매의향"
출력: 64점

→ 이 사람이 이 제품의 구매의향에 몇 점을 줄지 예측
→ 0.01초 만에 계산, GPU 없이 CPU로 동작
```
#### 어떻게 만들었나
**1단계: 학습 데이터 수집 (Knowledge Distillation)**
대형 AI(Claude)에게 "이 페르소나가 이 제품을 보면 어떻게 평가할까?"를
4,327번 질문해서 답변을 수집했습니다.
```
질문 예시:
  "20대 여성 대학생(가성비 중시, SNS 활발)이
   프리미엄 유기농 샐러드 배달 서비스(월 89,000원)를 보면
   호감도, 구매의향, 가격적절성을 각각 몇 점으로 평가할까?"
Claude 답변:
  호감도 73점, 구매의향 64점, 가격적절성 45점
  (이유: "건강에 관심은 있지만 대학생에게 월 9만원은 부담...")
```
이런 질문-답변 쌍을 4,327건 모았습니다.
- 16개 카테고리 (식품, 광고, 앱, 정책, 교육 등)
- 124개 자극물 (실제 제품/서비스/광고 시나리오)
- 40명의 다양한 페르소나 (10대~60대, 남녀, 직업 20종)
- 10개 카테고리의 설문 질문 94개
이 과정을 **"Knowledge Distillation (지식 증류)"**이라 합니다.
대형 AI의 판단 능력을 작은 모델로 옮기는 기법입니다.

**2단계: 패턴 추출 (임베딩 학습)**
수집된 데이터에서 규칙을 찾아냅니다.
```
데이터에서 발견되는 패턴:
  "가성비 중시" 특성 → 비싼 제품에 점수 낮음
  "건강 관심" 특성 → 건강 관련 제품에 점수 높음
  "20대" → 디지털 제품에 점수 높음, 보험에 점수 낮음
  "60대+" → 앱 서비스에 점수 낮음, 건강식품에 점수 높음
```

이 패턴을 수학적 모델로 학습합니다.

```
페르소나의 각 속성(연령, 성별, 직업, 성격)을
숫자 벡터(수치 목록)로 변환 → "임베딩"

  "20대" → [0.8, 0.3, 0.1, ...]  (24개 숫자)
  "가성비 중시" → [0.9, 0.1, 0.7, ...]  (24개 숫자)

제품/질문의 카테고리도 숫자 벡터로 변환

  "식품" → [0.3, 0.6, 0.4, ...]  (24개 숫자)
  "구매의향" → [0.4, 0.8, 0.2, ...]  (24개 숫자)

페르소나 벡터 + 제품 벡터 → 신경망 계산 → 점수 예측
```

**3단계: 학습 결과**

| 항목 | 수치 |
|------|------|
| 모델 크기 | 104KB (스마트폰 사진 1장보다 작음) |
| 파라미터 수 | 22,862개 |
| 학습 데이터 | 4,327건 |
| 평균 예측 오차 | 8.7점 (100점 만점 기준) |
| 추론 속도 | 0.01초 (GPU 없이) |

#### 왜 직접 만들었나

```
대형 AI(ChatGPT, Claude)에 매번 물어보면:
  → 질문당 비용 발생 (매번 과금)
  → 응답 시간 3~7초 (느림)
  → 외부 서비스 의존 (장애 시 동작 불가)

자체 임베딩 모델:
  → 비용 0원 (자체 서버에서 실행)
  → 0.01초 응답 (300배 빠름)
  → 외부 의존 없음 (독립적 동작)
```

### 2. 경량 LLM (답변 생성 모델)

#### 모델이 하는 일

```
입력: "20대 여성 대학생" + "유기농 샐러드" + "구매의향 64점"
출력: "건강에 관심은 있지만 대학생에게 월 9만원은 좀 부담이에요"

→ 임베딩 모델이 예측한 점수에 맞는 자연어 답변을 생성
→ CPU에서 3~10초로 동작
```

#### 어떻게 만들었나

**1단계: 베이스 모델 선택**

오픈소스 AI 모델인 Qwen 2.5-0.5B를 선택했습니다.
- 5억 개 파라미터 (GPT-4 대비 1/1000 수준)
- 한국어 이해 능력 보유
- CPU에서도 실행 가능한 경량 모델

**2단계: 파인튜닝 (추가 학습)**

임베딩 모델과 동일한 4,327건 데이터를 활용하되,
형태를 변환했습니다.

```
학습 데이터 형태:

  [태스크 1] 평가 이유 생성
  입력: 페르소나 정보 + 자극물 + "구매의향 64점"
  출력: "건강에 관심은 있지만 가격이 부담돼요"

  [태스크 2] 설문 답변
  입력: 페르소나 정보 + 질문 + 보기 + "선호도 78점"
  출력: {"선택": "그렇다", "이유": "출퇴근 시간이 아까워서..."}
```

LoRA(Low-Rank Adaptation) 기법으로 효율적으로 학습:
- 전체 모델을 다시 학습하지 않고, 핵심 부분만 조정
- RTX 4070 Ti GPU에서 약 1시간 소요
- 학습 비용: 전기세 수준 (추가 비용 없음)

**3단계: 배포**

학습된 모델을 클라우드 서버(GCP Cloud Run)에 배포:
- CPU 환경에서 동작 (GPU 불필요)
- 메모리 2GB로 운영
- 응답 시간: 3~10초

#### 두 모델의 협업 구조

```
질문/자극물이 들어오면:

  ① 임베딩 모델 (의사 역할)
     → "이 사람은 이 제품에 64점을 줄 것이다" (판단)
     → 0.01초

  ② 경량 LLM (간호사 역할)
     → "64점인 이유: 가격이 부담돼서입니다" (설명)
     → 3~10초

  = 판단은 자체 모델이, 표현은 자체 AI가
  = 외부 API 의존 없이 완전 독립 동작
```
